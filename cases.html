数据看板作品集

以下为日常商业化广告运营中沉淀的 5 个核心数据看板与自建分析工具的脱敏复刻，呈现分析框架、指标口径与方法论。

脱敏说明：本页所有客户 / 代理商名称、内部数据表名、看板地址与具体数值均已做脱敏或示意化处理（客户以 A–H 代称、代理商以甲乙丙代称、数值为代表性示意值），仅用于展示数据分析能力与看板设计思路，不代表任何企业真实经营数据。
01

商业化数据分析看板 · 多维下钻日报

数据看板 ｜ 大盘 → 行业 → 代理商 → 客户 四级下钻 ｜ 数据源：广告投放主表
背景

服务 60+ 客户、年消耗规模 3 亿+，每天需在海量投放数据中快速定位异常并给出归因，人工逐维度看数效率低、易漏判。

动作

搭建「大盘 → 行业 → 代理商 → 客户」四级下钻看板，统一 效率 CPM / 超成本 / CTCVR / 曝光比 DAU 口径，引入双基线（前 1 日 + 前 7 日均值）异常检测，波动 >10% 自动触发下钻。

成果

日报从「罗列数据」升级为「现象 → 根因 → 建议动作」三段式结论，TOP3 异常一屏定位；沉淀为部门标准 SOP。

02

广告全链路漏斗 & eCPM 过滤看板

数据看板 ｜ 请求 → 出广告 → 曝光 → 转化全链路 ｜ 数据源：实时日志 + 投放主表
背景

收入波动时，需快速判断到底是「请求少了、过滤多了，还是转化差了」，单看曝光 / 收入无法定位漏损环节。

动作

把「请求 → 算法出广告 → 最终出广告 → 曝光 → 点击 → 转化」做成全链路漏斗，叠加填充率与低 eCPM 过滤分层，用 pqtid 作精确请求分母。

成果

定位漏损集中在「有曝光无点击层」（eCPM 过滤占比最高 10%）；7 日填充率 67.7%→71.5% 持续改善，为算法侧优化提供量化依据。

广告请求 → 转化 全链路漏斗示意数据 · 已脱敏
① 广告请求（pqtid）9,980,000
② 算法判定出广告9,180,000 92.0%
③ 最终出广告8,010,000 80.3%
④ 曝光（填充率 71.5%）7,140,000 71.5%
⑤ 点击（CTR 6.0%）428,000
⑥ 转化（CTCVR 1.5%）107,000

口径：填充率 = 曝光次数 / 请求数(count distinct pqtid)；每个 pqtid 一对一映射一次请求，是精确的请求分母，避免用曝光 UV 误算。

洞察：过滤并非集中在「无曝光首层」，而是「有曝光无点击层」——这部分用户近期看广告但不点击，模型预估 CTR 极低 → eCPM 极低 → 被过滤。是否首层不是过滤主因。

03

广告变更日志看板 · 调价方向与批量停投检测

数据看板 ｜ 操作行为归因 ｜ 数据源：投放操作日志
背景

每天数千条操作日志（调价 / 开停投 / 定向变更），按时间或次数排序无法看出哪些操作真正影响了大盘。

动作

建立「调价方向三部曲」（提降比 / 降幅分布 / 极端事件），叠加批量操作检测（单分钟停投≥5 次）与停投恢复跟踪，统一按消耗影响面倒序。

成果

用提降比量化市场竞争力变化；识别 17:00–18:00 批量停投高峰，并测算停投释放曝光的 CPM 差价损失。

04

实时 DAU / 流量质量看板

数据看板 ｜ 5 分钟实时监控 + 归因决策 ｜ 数据源：实时日志
背景

收入下滑时需立刻判断是「用户量少了」还是「人均曝光频次低了」，否则容易误判优化方向。

动作

定义并实时监控 广告DAU / AD PV·DAU / ARPU / 曝光成功率 四大健康度指标，配套归因决策树一键定位根因。

成果

异常时 5 分钟内定位「量 vs 频次 vs 效率」问题归属，把模糊的「收入跌了」转化为可执行的单一根因。

05

自建广告数据取数工具 & 财务对账模块

自建工具 ｜ Python(Flask) + 单文件前端 ｜ 替代通用 BI 面板的 ad-hoc 取数 REPL
背景

通用 BI 表格面板渲染 20+ 列 × 数千行经常卡崩，维度 / 指标靠链式下拉迭代极慢；月度财务对账长期靠人工 Excel，易错且耗时。

动作

独立开发键盘流取数工具：复用浏览器登录态走 BI 代理查询数仓，55 维度 + 32 指标 + 33 衍生比率口径与原面板 1:1 对齐；并切出财务对账子模块，资金流水自动对账并对齐 Excel 模板。

成果

取数从「点选下拉」变为「敲快捷键」，端到端体验显著提速；财务对账月度结果与人工模板完全吻合；沉淀多条业务铁律（审批状态过滤、映射去重防 25× 放大）。

口径 1:1 对齐

55 维度 / 32 指标 / 33 衍生比率，含企微切换日 case-when，改一处常量联动 5 个加微指标。

字段时效探测

一条大 SQL 探测各列最早/最新有数日期，ETL 漏刷一眼可见，避免空数据误判。

模糊匹配过滤

LIKE ANY / REGEXP 解决简称的异形字 / 形近字（如「涛↔濤」）匹配难题。

性能可量化

状态条实时显示 p50 / p95 查询延迟分位，量化「快了多少」。
