1AI / 数据产品实践

现任期间（太字流动）自建 / 主导，以 AI 为协作者落地。

数分机器人 · 广告数据日报分析（规则引擎 + LLM 归因）

自建 · 设计 + 实现 · 团队 3 人日常使用 · 迭代 2 个月 · human-in-the-loop
  • 问题：每天人工撰写广告异常归因日报约 30 分钟，重度依赖个人经验，难规模化、难传承
  • 方案：脚本自数仓抓取 8 天大盘 / 行业 / 客户数据 → 按规则做阈值异常检出与结构 / 效率效应分解 → 拼装含「现象 → 根因 → 建议」三段式与归因红线的 prompt → 交 LLM 做深度归因
  • 防幻觉设计：所有数值均由数仓计算后输入、模型不做算术，只负责归因叙述，从架构上规避数值幻觉
  • 成果：1 周产出、迭代近 2 个月；团队 3 人日常使用；日报从人工 30 分钟压缩为分钟级复核
  • 已知边界：异常检出可靠、明显异常即时发现；归因质量尚未量化，当前作人工复核的辅助
配套 Eval 评测（已建）：为它建了一套自动化评测 —— 22 例历史日报黄金集，代码型指标校验数值忠实率与异常召回 / 精确，LLM-as-judge 评归因质量与机制编造率，并作 prompt 改动的回归门禁（含 with / without skill A/B）。▶ 查看 Eval Demo

  • 定义口径：梳理对账所需字段与业务口径，对齐对账逻辑与计算逻辑
  • 打通链路 + 可运营：将对账字段与数仓 / 看板打通，实现数据可监控、可上报、可排查，把一次性核账变为常态化机制
